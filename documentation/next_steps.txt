# THE BIG PICTURE


**What we are building:**
A three-agent system that automates financial risk screening of SEC 10-K filings. An analyst types a company name and gets a cited risk report in seconds instead of reading 200 pages across 5 years.


**Why it matters:**
Single-agent RAG systems fail on 81% of financial QA tasks (FinanceBench). Our system separates data fetching, risk extraction, and comparison into three specialized agents — each solving a specific failure mode of naive RAG.


**One-line pitch:**
*"We built an agentic system that does in 30 seconds what takes a financial analyst 3 days — and every claim it makes can be verified in the original filing."*


---


# SYSTEM OVERVIEW


## Agent 1 — Fetcher ✅ COMPLETE
**Owner: Sharan**
Connects to SEC EDGAR API and pulls 5 years of 10-K filings per company. Outputs raw HTML sections and cleaned XBRL financial data.


## Agent 2 — Curator
**Owner: Sharan**
Takes Agent 1 output and produces a structured profile per company per year. Contains financial deltas and risk signals with citations. Saves as one JSON file per filing year.


## Agent 3 — Comparison Agent
**Owner: Om**
Takes a user-queried company, matches it to similar companies in the database using embeddings, and generates a cited analyst report showing what happened to matched companies and what trajectory the queried company may be heading toward.


## Baseline RAG + Evaluation
**Owner: Sumit**
Builds a simple single-agent RAG pipeline for comparison. Measures how much better FinAgent performs on citation accuracy and hallucination rate.

# SIGNAL SCHEMA — LOCKED


This is the contract between all agents. Do not change it after Day 2.


## File naming
`{ticker_lowercase}_{year}.json`
Examples: snap_2023.json, aapl_2022.json, we_2021.json


## Full file structure


```json
{
  "company": "SNAP",
  "ticker": "SNAP",
  "filing_year": 2023,
  "financial_deltas": {
    "Revenue": {"value": -0.05, "label": "moderate_decline"},
    "NetIncome": {"value": -0.23, "label": "severe_decline"},
    "Cash": {"value": -0.41, "label": "severe_decline"},
    "Assets": {"value": -0.08, "label": "moderate_decline"},
    "LongTermDebt": {"value": 0.12, "label": "moderate_growth"},
    "OperatingCashFlow": {"value": -0.67, "label": "severe_decline"},
    "ResearchAndDevelopment": {"value": 0.03, "label": "stable"},
    "GrossProfit": {"value": -0.11, "label": "moderate_decline"}
  },
  "risk_signals": [
    {
      "signal_id": 1,
      "topic": "liquidity_risk",
      "signal_type": "cash_runway_concern",
      "section": "Item 7",
      "filing_year": 2023,
      "company": "SNAP",
      "summary": "Operating cash outflows widened year over year with no disclosed path to breakeven.",
      "severity": "high",
      "citation": "SNAP 10-K 2023, Item 7"
    }
  ],
  "embedding_text": "Company: SNAP, Year: 2023. Revenue: moderate_decline. NetIncome: severe_decline ...",
  "embedding_vector": [0.021, -0.034, "... 1536 values"]
}
```


## Financial delta severity buckets


| Delta Range | Label |
|---|---|
| Above +20% | strong_growth |
| +5% to +20% | moderate_growth |
| -5% to +5% | stable |
| -20% to -5% | moderate_decline |
| Below -20% | severe_decline |

`liquidity_risk` | `market_risk` | `legal_risk` | `operational_risk` | `macro_risk` | `strategic_risk`

# RISK SIGNAL TAXONOMY


The LLM extracts every signal matching these types. No fixed count. One signal per distinct risk instance.


## Liquidity and Financial Health
*Primary sections: Item 7, Item 7A*


- **cash_runway_concern** — cash burn, operating losses, insufficient cash to fund operations
- **going_concern_warning** — explicit going concern language or auditor doubt
- **debt_covenant_risk** — risk of breaching loan covenants, waiver requests
- **refinancing_risk** — debt maturing soon with no clear refinancing path
- **negative_operating_cash_flow** — sustained cash outflow from core operations
- **working_capital_deficit** — current liabilities exceeding current assets
- **dividend_buyback_suspension** — shareholder returns suspended due to cash constraints
- **impairment_charges** — goodwill or asset write-downs from prior acquisitions
- **revenue_concentration_risk** — heavy dependence on one or few customers
- **accounts_receivable_deterioration** — rising DSO or bad debt provisions


## Market and Competitive Risk
*Primary sections: Item 1, Item 1A*


- **intensifying_competition** — new entrants, pricing pressure, market share loss
- **product_obsolescence** — core product becoming outdated or commoditized
- **platform_dependency** — reliance on Apple, Google, AWS, or other third-party platforms
- **customer_churn_risk** — difficulty retaining users or subscribers
- **pricing_power_erosion** — inability to raise prices, margin compression from discounting
- **market_saturation** — slowing TAM or penetration ceiling reached
- **network_effect_weakening** — declining engagement or user growth signals
- **advertiser_concentration** — dependence on few advertisers for significant revenue
- **subscription_model_risk** — churn, downgrade, or cancellation trends
- **geographic_concentration** — heavy revenue dependence on one region or country


## Legal, Regulatory and Compliance Risk
*Primary sections: Item 1A, Item 1C, Item 3*


- **active_litigation** — pending lawsuits, class actions, arbitration with material exposure
- **regulatory_investigation** — FTC, SEC, DOJ, or foreign regulator investigations
- **antitrust_exposure** — risk of forced breakup, divestiture, or conduct restrictions
- **data_privacy_violation** — GDPR, CCPA, or other privacy regulation violations
- **cybersecurity_incident** — breaches, unauthorized access, ransomware disclosed
- **export_control_risk** — exposure to restricted countries or entities
- **ip_infringement_claim** — active patent, copyright, or trade secret litigation
- **content_moderation_liability** — legal exposure from user-generated content
- **environmental_compliance** — regulatory fines related to environmental standards
- **government_contract_risk** — debarment, contract cancellation, or compliance failures


## Operational Risk
*Primary sections: Item 1, Item 1A, Item 7*


- **key_person_dependency** — reliance on a small number of executives or founders
- **talent_retention_risk** — difficulty hiring or retaining engineers or executives
- **supply_chain_concentration** — sole-source suppliers, semiconductor shortages
- **infrastructure_single_point_failure** — single points of failure in cloud or on-premise
- **cybersecurity_vulnerability** — ongoing or systemic security weaknesses disclosed
- **third_party_vendor_dependency** — over-reliance on external vendors for critical services
- **manufacturing_defect_risk** — quality control issues, product recalls, yield problems
- **scaling_risk** — inability to scale operations to meet demand
- **disaster_recovery_gaps** — insufficient backup or recovery capabilities
- **internal_controls_weakness** — material weakness or significant deficiency in reporting


## Macroeconomic and Geopolitical Risk
*Primary sections: Item 1A, Item 7A*


- **interest_rate_sensitivity** — floating rate debt or portfolio exposed to rate changes
- **foreign_currency_exposure** — unhedged revenue or costs in multiple currencies
- **inflation_impact** — rising input costs, wage inflation, margin pressure
- **recession_sensitivity** — discretionary product or ad revenue exposed to downturns
- **china_geopolitical_exposure** — revenue, manufacturing, or supply chain in China
- **trade_tariff_risk** — exposure to tariffs on hardware, components, or services
- **sanctions_exposure** — customers or partners in sanctioned jurisdictions
- **sovereign_debt_risk** — operations or receivables in fiscally unstable countries


## Strategic and Growth Risk
*Primary sections: Item 1, Item 7*


- **failed_acquisition** — integration challenges, acquisition impairment, dilutive M&A
- **new_market_entry_failure** — expansion into new geography or segment underperforming
- **rd_productivity_risk** — high R&D spend with unclear or declining returns
- **moonshot_unproven_bet** — capital allocated to speculative new businesses
- **business_model_transition_risk** — pivot between models with execution risk
- **customer_adoption_lag** — new product or feature with slower than expected uptake
- **monetization_uncertainty** — product has users but no clear revenue model
- **strategic_partnership_dependency** — critical outcomes tied to one or two partners
- **competitive_moat_erosion** — prior differentiators weakening
- **capital_allocation_risk** — buybacks or dividends inconsistent with financial health


### Severity rules
- **High** — explicit financial impact disclosed, active investigation, going concern language, material weakness
- **Medium** — risk acknowledged with potential impact, trend worsening but not critical
- **Low** — standard boilerplate risk disclosure with no specific financial impact stated

# LLM PROMPT TEMPLATES


## Agent 2 — Signal Extraction Prompt


```
You are a financial risk analyst extracting structured risk signals
from a SEC 10-K filing section.


COMPANY: {company}
TICKER: {ticker}
FILING YEAR: {filing_year}
SECTION: {section_name}


RISK TAXONOMY:
{paste full taxonomy here}


TEXT:
{section_text}


Extract every risk signal from this text that matches a type in the
taxonomy above. Each signal must be unique — no duplicate themes.


Return ONLY a JSON array. No preamble, no explanation, no markdown.


Each object must follow this exact schema:
{
  "signal_id": <integer>,
  "topic": <topic value from taxonomy>,
  "signal_type": <signal_type value from taxonomy>,
  "section": "{section_name}",
  "filing_year": {filing_year},
  "company": "{company}",
  "summary": <1-2 sentence description of the risk in plain English>,
  "severity": <"high", "medium", or "low" per severity rules>,
  "citation": "{company} 10-K {filing_year}, {section_name}"
}


Rules:
- Only extract signals grounded in specific language from the text
- Do not infer risks not explicitly disclosed
- If no signals match, return an empty array
- Do not fabricate signals to pad the count
```


## Agent 3 — Analyst Report Prompt


```
You are a senior financial risk analyst preparing a briefing.
You must cite every claim.


QUERIED COMPANY: {company_n}
FILING YEAR: {query_year}


FINANCIAL COMPARISON
{financial_comparison_table}


QUERIED COMPANY RISK SIGNALS
{company_n_signals}


MATCHED CASE LIBRARY
{matched_companies_with_signals_and_lookahead}


Generate a structured analyst report with exactly these four sections:


1. FINANCIAL POSITION
   Compare {company_n} financial trajectory to matched peers.
   Cite every metric reference. Format: [COMPANY 10-K YEAR, Section]


2. KEY RISK SIGNALS
   Summarize the top risk themes for {company_n}.
   Cite every claim.


3. HISTORICAL PARALLELS
   For each matched company explain what risks materialized,
   which were averted, and how. Cite every filing reference.


4. POTENTIAL TRAJECTORY
   Based on historical parallels, what are 2-3 likely trajectories
   for {company_n}? Ground every claim in cited evidence.


Rules:
- Never make a claim without a citation
- Never introduce information not in the provided data
- If evidence is insufficient, say so explicitly
- Keep each section under 200 words
```

# BASELINE RAG PIPELINE


**Owner: Sumit**


This is the comparison system. Keep it intentionally simple — no schema, no citations, no structure.


**Pipeline:**
1. Take raw 10-K HTML sections — same input as FinAgent
2. Chunk into 512-token blocks with 50-token overlap
3. Embed using text-embedding-3-large — same model as Agent 3 for fair comparison
4. Store in FAISS flat index in memory
5. On query: embed query, retrieve top 5 chunks, pass to Claude Sonnet


**Prompt — no structure, no citation instruction:**
```
You are a financial analyst. Based on the following excerpts
from SEC filings, answer this question: {query}


EXCERPTS:
{retrieved_chunks}


Answer:
```


**Expected failure modes — these are your results:**
- Retrieves boilerplate chunks instead of risk-relevant ones
- Generates claims with invented or missing citations
- Misses numerical signals because numbers do not embed well
- Cannot answer Query 3 at all because it has no case library

# EVALUATION FRAMEWORK


**Owner: Sumit**


## Three standard queries — run on both systems


- **Query 1** — "What are the top risk factors for {company} in {year}?"
- **Query 2** — "How has {company}'s financial position changed over the past 3 years?"
- **Query 3** — "What trajectory is {company} heading toward based on historical parallels?"


## Metric 1 — Citation accuracy (automated)


For each signal in an Agent 2 output file, extract key noun phrases from the summary field and search for them in the raw section HTML. Score each signal as:
- **Grounded** — phrases found in source text
- **Partially grounded** — some phrases found
- **Hallucinated** — no basis in source text


Report as a percentage across all signals for one company.


## Metric 2 — Hallucination rate (manual)


Pick 20 signals from one company output. Read each cited section in the raw filing. Grade each signal as:
- **Faithful** — summary accurately reflects source text
- **Exaggerated** — overstates severity beyond what text supports
- **Fabricated** — no basis in source text


Formula: `hallucination_rate = (exaggerated + fabricated) / 20`


## Metric 3 — Retrospective validation (strongest result)


Run Agent 3 on WeWork 2020 or 2021 filing — one to two years before their bankruptcy filing. Ask: did the system flag the right risks? Did matched companies point toward the correct trajectory? This is your backtesting result and the core of your paper's argument.

|---|---|---|

| Hallucination rate | % | % |
| Financial delta accuracy | % | % |
| Retrospective risk flag accuracy | % | % |
| Query 3 answerability | No | Yes |
